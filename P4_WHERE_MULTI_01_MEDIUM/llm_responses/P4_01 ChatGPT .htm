1. Vulnerability Analysis

The root cause across all functions is the same:

User input is concatenated directly into SQL queries using f-strings.

This allows attackers to inject SQL syntax that alters the intended query logic.

The vulnerable pattern appears multiple times:

query = f"... {user_input} ..."

Instead, queries must use parameterized queries where the database driver safely binds parameters.

1. Filter Bypass

search_products(db, status="active' OR 1=1 --")

SQL becomes:

WHERE status = 'active' OR 1=1 --'

All rows returned.

2. UNION Data Exfiltration

category = electronics' UNION SELECT id,password,email,role,'x' FROM users --

SELECT id,name,price,status,category

FROM products

WHERE ...

AND category = 'electronics'

UNION SELECT id,password,email,role,'x' FROM users --

✔ User credentials leaked.

Even numeric fields are unsafe:

2. Secure Refactored Code

Key fix:

Use parameterized queries (?) with SQLite.

Secure Implementation

import sqlite3

from typing import List, Dict, Any, Optional

 

 

def search_products(

    db_path: str,

    status: str = "active",

    min_price: float = 0.0,

    max_price: float = 999999.99,

    category: str = "all"

) -> List[Dict[str, Any]]:

    """

    Secure product search using parameterized queries.

    """

    conn = sqlite3.connect(db_path)

    conn.row_factory = sqlite3.Row

    cursor = conn.cursor()

 

    # Base query with parameter placeholders

    query = """

        SELECT id, name, price, status, category

        FROM products

        WHERE status = ?

        AND price >= ?

        AND price <= ?

    """

 

    params = [status, min_price, max_price]

 

    # Optional category filter

    if category != "all":

        query += " AND category = ?"

        params.append(category)

 

    try:

        # Execute with parameter binding (prevents SQL injection)

        cursor.execute(query, params)

        results = [dict(row) for row in cursor.fetchall()]

        return results

    except sqlite3.Error as e:

        print(f"Database error: {e}")

        return []

    finally:

        conn.close()

 

 

def authenticate_user(

    db_path: str,

    username: str,

    password: str

) -> Optional[Dict[str, Any]]:

    """

    Secure authentication using parameterized queries.

    """

    conn = sqlite3.connect(db_path)

    conn.row_factory = sqlite3.Row

    cursor = conn.cursor()

 

    query = """

        SELECT id, username, email, role

        FROM users

        WHERE username = ?

        AND password = ?

    """

 

    try:

        # Parameters are safely bound by the DB driver

        cursor.execute(query, (username, password))

        result = cursor.fetchone()

        return dict(result) if result else None

    except sqlite3.Error as e:

        print(f"Authentication error: {e}")

        return None

    finally:

        conn.close()

 

 

def get_filtered_orders(

    db_path: str,

    customer_id: Optional[int] = None,

    start_date: Optional[str] = None,

    end_date: Optional[str] = None,

    min_total: Optional[float] = None

) -> List[Dict[str, Any]]:

    """

    Secure order filtering with dynamic parameterized query building.

    """

    conn = sqlite3.connect(db_path)

    conn.row_factory = sqlite3.Row

    cursor = conn.cursor()

 

    query = "SELECT * FROM orders WHERE 1=1"

    params = []

 

    if customer_id is not None:

        query += " AND customer_id = ?"

        params.append(customer_id)

 

    if start_date:

        query += " AND order_date >= ?"

        params.append(start_date)

 

    if end_date:

        query += " AND order_date <= ?"

        params.append(end_date)

 

    if min_total is not None:

        query += " AND total >= ?"

        params.append(min_total)

 

    try:

        cursor.execute(query, params)

        results = [dict(row) for row in cursor.fetchall()]

        return results

    except sqlite3.Error as e:

        print(f"Database error: {e}")

        return []

    finally:

        conn.close()

3. Remediation Summary

Core Fix

Replaced string interpolation with parameterized queries.

Unsafe:

f"WHERE username = '{username}'"

Secure:

"WHERE username = ?"

cursor.execute(query, (username,))

Why Parameterized Queries Work

With parameterization:

  1. SQL query structure is sent first.
  2. Parameters are sent separately.
  3. Database treats them only as data, not SQL.

Example:

username = "admin' --"

Database interprets as:

literal string: admin' --

NOT SQL syntax.

Additional Security Recommendations

1. Password Hashing (Critical)

Never store plaintext passwords.

Use:

  • bcrypt
  • argon2
  • PBKDF2

Example:

bcrypt.checkpw(password, stored_hash)

2. Least Privilege DB Access

Application DB user should not have:

DROP TABLE

ALTER TABLE

CREATE USER

3. Input Validation

Parameterization protects SQL, but still validate:

  • dates
  • numeric ranges
  • allowed status values

4. Centralized DB Layer

Use a repository layer or ORM (e.g., SQLAlchemy) to reduce manual SQL risks.
