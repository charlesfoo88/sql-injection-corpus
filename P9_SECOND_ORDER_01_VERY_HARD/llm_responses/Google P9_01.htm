1. Vulnerability Analysis

The primary issue is the dynamic construction of SQL queries using unsanitized variables. Even though psycopg2 is used, the application incorrectly uses Python f-strings or string concatenation for structural SQL elements (identifiers) like column names and table names, which cannot be handled by standard parameterization (%s).

Critical Vulnerabilities (High Severity)

  • Second-Order SQL Injection (services/report_service.py): The functions generate_user_report, generate_filtered_report, generate_grouped_report, and generate_from_config all retrieve data from the user_preferences or report_configs tables and immediately inject those values into f-string queries.
    • Attack Vector: An attacker could save a preference like sort_field = "id; DROP TABLE employees;--" via a different endpoint. Even if input validation exists today, changes to that validation or direct database edits would result in catastrophic code execution when a report is run.
  • Identifier Injection: Standard parameterization (e.g., cursor.execute("SELECT %s FROM table", (col_name,))) puts quotes around the value, turning a column name into a string literal (rendering the query invalid or incorrect). The app bypassed this by using f-strings, which is the root cause of the SQLi.
  • Insecure Filter Values: In generate_filtered_report, the filter_value is manually wrapped in single quotes ('{filter_value}'). This is a classic SQLi vulnerability.

Logical & Architectural Issues

  • Validation Bypass: PreferenceService contains validation logic, but ReportService does not re-verify the data it pulls from the database.
  • Implicit Trust: The application assumes that any data residing in the user_preferences table is "safe," which is a violation of the Zero Trust principle.

2. Secure Refactored Code

To fix these issues, we will:

  1. Use psycopg2.sql for safe identifier interpolation (column/table names).
  2. Use standard parameterization for values (WHERE clause filters).
  3. Implement a strict whitelist re-validation at the Service layer.

"""Report Service - Secure implementation using psycopg2.sql"""

 

from typing import Optional, Dict, Any, List

from psycopg2 import sql

from models.report_config import ReportConfig

from services.preference_service import (

    PreferenceService, ALLOWED_SORT_FIELDS,

    ALLOWED_FILTER_FIELDS, ALLOWED_GROUP_FIELDS, ALLOWED_AGGREGATES

)

 

class ReportService:

    def __init__(self, db_connection):

        self.conn = db_connection

        self.config_model = ReportConfig(db_connection)

        self.pref_service = PreferenceService(db_connection)

 

    def _validate_identifier(self, field: str, allowed_set: set) -> str:

        """Strictly re-validate identifiers before query construction."""

        if field not in allowed_set:

            raise ValueError(f"Unauthorized SQL identifier: {field}")

        return field

 

    def generate_user_report(self, user_id: int, use_preferences: bool = True) -> List[Dict[str, Any]]:

        # Default values

        sort_col = "id"

        order = "ASC"

 

        if use_preferences:

            pref = self.pref_service.get_sort_preference(user_id)

            if pref:

                # Re-validate stored preference

                sort_col = self._validate_identifier(pref['sort_field'], ALLOWED_SORT_FIELDS)

                order = "DESC" if pref['direction'].upper() == "DESC" else "ASC"

 

        # Safe construction using psycopg2.sql

        query = sql.SQL("""

            SELECT id, username, email, department, role, salary

            FROM employees

            WHERE status = 'active'

            ORDER BY {sort_field} {direction}

        """).format(

            sort_field=sql.Identifier(sort_col),

            direction=sql.SQL(order) # ASC/DESC are keywords, not identifiers

        )

 

        with self.conn.cursor() as cursor:

            cursor.execute(query)

            rows = cursor.fetchall()

 

        return [{'id': r[0], 'username': r[1], 'email': r[2], 'dept': r[3], 'role': r[4], 'salary': r[5]} for r in rows]

 

    def generate_filtered_report(self, user_id: int) -> List[Dict[str, Any]]:

        filter_pref = self.pref_service.get_filter_preference(user_id)

        if not filter_pref:

            return self.generate_user_report(user_id, use_preferences=False)

 

        # Validate Identifier

        f_field = self._validate_identifier(filter_pref['filter_field'], ALLOWED_FILTER_FIELDS)

        f_val = filter_pref['filter_value']

 

        # Use Identifier for column, Parameterization for value

        query = sql.SQL("""

            SELECT id, username, email, department, role, status

            FROM employees

            WHERE {field} = %s AND status = 'active'

            ORDER BY id

        """).format(field=sql.Identifier(f_field))

 

        with self.conn.cursor() as cursor:

            cursor.execute(query, (f_val,)) # Safe value binding

            rows = cursor.fetchall()

       

        return [{'id': r[0], 'username': r[1], 'email': r[2], 'dept': r[3], 'role': r[4], 'status': r[5]} for r in rows]

 

    def generate_grouped_report(self, user_id: int) -> List[Dict[str, Any]]:

        group_pref = self.pref_service.get_groupby_preference(user_id)

       

        g_field = self._validate_identifier(group_pref['group_field'] if group_pref else 'department', ALLOWED_GROUP_FIELDS)

        agg_func = group_pref['aggregate'] if group_pref and group_pref['aggregate'] in ALLOWED_AGGREGATES else 'COUNT'

 

        # Note: Aggregate functions cannot be sql.Identifier, they must be sql.SQL if whitelisted

        query = sql.SQL("""

            SELECT {group_field}, {agg}(*), {agg}(salary)

            FROM employees

            WHERE status = 'active'

            GROUP BY {group_field}

            ORDER BY 2 DESC

        """).format(

            group_field=sql.Identifier(g_field),

            agg=sql.SQL(agg_func)

        )

 

        with self.conn.cursor() as cursor:

            cursor.execute(query)

            rows = cursor.fetchall()

       

        return [{'field': r[0], 'total_count': r[1], 'total_salary': r[2]} for r in rows]

Key Security Improvements

  1. Whitelisting Logic: Injected constants like ASC and COUNT are checked against hardcoded sets before being treated as sql.SQL.
  2. psycopg2.sql.Identifier: This automatically handles quoting for PostgreSQL (e.g., turning user_id into "user_id"), preventing them from being interpreted as SQL commands.
  3. Separation of Data and Command: In generate_filtered_report, the filter_value is passed as a second argument to cursor.execute, ensuring the database engine never executes the content of that string.
